# Scene Description Guide

## Spatial Vocabulary

Use consistent spatial language so the user builds an accurate mental model.

### Horizontal
- "on the left" / "on the right" — from the camera's point of view
- "in the centre" / "centred in frame"
- "toward the left edge" / "partially out of frame on the right"

### Depth
- "in the foreground" — closest objects (< 0.5m)
- "in the mid-ground" — 0.5m–1.5m
- "in the background" — > 1.5m, or behind main subject

### Vertical
- "on the surface" / "on the table" / "on the floor"
- "elevated" / "mounted on the wall" / "overhead"
- "below the camera's view" — if something is likely cut off at the bottom

## Object Description Pattern

For each notable object:
1. Name/type ("a red plastic cup")
2. Location ("on the left side of the table")
3. Distance if measurable ("approximately 0.6m away")
4. Notable state ("upright", "upside down", "open", "closed")

Example: "A red 2×4 Lego brick is on the left side of the table, approximately 0.4 metres away, lying flat."

## Lighting Callouts

If lighting is notably relevant:
- "The scene is well-lit"
- "Low light conditions — colours may be inaccurate"
- "There is strong backlighting — some detail is lost"
- "Shadows obscure the right side of the frame"

## Safety-Relevant Objects

Always explicitly mention:
- People or faces in frame
- Hands or limbs near the robot's work area
- Liquids or fragile objects
- Power cables or wires on the floor
